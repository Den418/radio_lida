# 📻 Радио Лида — RadioPlayerV3

Бот-центр управления интернет-радио «Радио Лида»: музыка, очередь,
баннеры и видео, предложения от слушателей и кликабельная админ-панель
в голосовых чатах Telegram.

Бот задуман как закрытый инструмент управления, а не паблик-сервис.
Обычные пользователи при `/start` видят только контакт поддержки и
подсказку про `/suggest`, всё управление доступно исключительно
администраторам.

---

## ⚡ Что исправлено в этой версии, по сравнению с оригиналом

| Проблема в оригинале | Исправление |
|---|---|
| `bot.run(main())` блокирует процесс, код ниже никогда не выполняется | Весь код перенесён в `async def main()` |
| `asyncio.run(main())` создавал второй, другой event loop, чем тот, что уже сидел внутри `PyTgCalls` → `Future attached to a different loop` | Loop создаётся явно в начале `main.py`, до импорта `utils.py`, и переиспользуется через `loop.run_until_complete()` |
| `Thread(target=func())` вызывает функцию сразу, а не передаёт её | Исправлено на `Thread(target=func)` |
| Два разных клиента бота в `utils.py` и `main.py` с одним токеном | Единый клиент в `bot_client.py` |
| Устаревший `pytgcalls` не ставится на Python 3.10+ и Windows | Переход на `py-tgcalls`, у него есть колёса под все платформы |
| `pyrogram.errors.GroupcallForbidden` отсутствует в новом Pyrogram | Используется `pyrofork`, поддерживаемый форк с тем же именем модуля |
| `filters.user(ADMINS)` замораживает список админов на момент запуска | Динамический `admin_filter`, проверяет права заново при каждом вызове |
| Панель падала с `MESSAGE_NOT_MODIFIED`, если нажать кнопку без изменений на экране | Общая функция `_render()` ловит эту ошибку и всё равно отвечает на нажатие |
| Если голосовой чат обрывался сам по себе, бот просто замолкал навсегда | Бот сам замечает обрыв и переподключается, плюс фоновый сторож на случай, если событие об обрыве вообще не пришло |

## 🛠 Требования

- Python 3.10+
- FFmpeg, для аудио и видео
- Отдельный аккаунт Telegram для юзербота, не основной

---

## 🚀 Установка и запуск

```bash
git clone https://github.com/AsmSafone/RadioPlayerV3
cd RadioPlayerV3
pip install -r requirements.txt
```

Установи FFmpeg, если его нет:
```bash
# Ubuntu/Debian
sudo apt install ffmpeg
# macOS
brew install ffmpeg
# Windows — https://ffmpeg.org/download.html
```

Получи строку сессии юзербота:
```bash
python3 gen_session.py
```

Настрой переменные окружения:
```bash
cp .env.example .env
# заполни .env любым редактором
```

Запусти:
```bash
python3 main.py
```

## ⚙️ Настройка

### Обязательные переменные

| Переменная | Как получить |
|---|---|
| `API_ID` / `API_HASH` | https://my.telegram.org, раздел API development tools |
| `BOT_TOKEN` | @BotFather, команда /newbot |
| `SESSION_STRING` | `python3 gen_session.py` |
| `CHAT_ID` | ID канала или группы, отрицательное число |

### Необязательные переменные

| Переменная | По умолчанию | Описание |
|---|---|---|
| `RADIO_NAME` | Радио Лида | Название, используется в приветствиях, /help, панели |
| `SUPPORT_CONTACT` | @Yakton_support | Контакт поддержки для обычных пользователей |
| `AUTH_USERS` | пусто | Доп. администраторы, ID или username через пробел |
| `STREAM_URL` | тестовое радио | URL радио-потока |
| `LOG_GROUP` | пусто | ID группы для логов и предложений от слушателей |
| `ADMIN_ONLY` | True | Только админы добавляют треки напрямую, бот закрытый по умолчанию |
| `BANNER_URL` | пусто | Картинка, гифка или видео вместо чёрного экрана |
| `BANNER_PINNED` | False | Если True, баннер не уступает обложкам треков |
| `RADIO_TITLE` | 📻 РАДИО ЛИДА... | Заголовок голосового чата во время радио |
| `MAXIMUM_DURATION` | 15 | Лимит длины трека в минутах |
| `DELAY` | 10 | Через сколько секунд удалять служебные сообщения |
| `EDIT_TITLE` | True | Менять заголовок голосового чата |

## 🎛 Панель управления

Главный способ управлять ботом, команда `/admin`. Открывается кликабельная
панель прямо в чате:

```
Панель «Радио Лида»

📻 Играет радио Радио Лида

[▶️ Играть]      [⏸ Пауза]
[⏭ Пропустить]   [⏹ Стоп]
[📻 Радио]       [💡 Предложения]
[📋 Очередь]     [🔊 Громкость 100%]
[🖼 Баннер]      [👮 Админы]
[📊 Статистика]  [🔄 Обновить]
[🔄 Перезапуск бота]
[✖️ Закрыть]
```

Все текстовые команды по-прежнему работают и никуда не делись, панель
просто даёт то же самое кнопками, без необходимости запоминать команды.

Админы видят панель и полную справку по `/start`. Все остальные видят
короткое сообщение с контактом поддержки и подсказкой про `/suggest`.

## 👮 Администраторы бота

Три источника прав, все действуют одновременно.

1. Администраторы чата, подхватываются из Telegram автоматически
2. `AUTH_USERS` в `.env`, статичный список, меняется только вручную
3. Добавленные через панель, хранятся в `admins.json`, переживают перезапуск

Управление динамическими админами:
- `/admin` → «Админы», список и кнопки удаления
- `/addadmin id_или_@username`, можно и ответом на сообщение человека
- `/removeadmin id`

Убрать можно только тех, кто добавлен через панель. Админов из `.env`
или самого чата бот менять не может, это осознанное ограничение.

## 🎙 Если хочешь говорить в эфире сам

Бот не транслирует микрофон, потому что если он будет работать на
сервере, у сервера физически нет звуковой карты, и функция просто не
будет работать.

Проще и надёжнее сделать так: зайди в тот же голосовой чат сам, своим
личным аккаунтом или от имени канала, и говори. Музыка бота будет играть
фоном, а если хочешь тишину на время разговора, нажми в панели ⏸ Пауза
или ⏹ Стоп, а потом ▶️ Играть, когда закончишь.

## 💡 Предложения от слушателей

Обычные пользователи не могут сами командовать ботом, зато могут
предложить трек или видео командой `/suggest`:

```
/suggest Imagine Dragons Believer
```

Или ответом на аудио, видео или голосовое сообщение. Можно и просто
скинуть боту файл в личку без всякой команды, если отправитель не
админ, файл тоже уйдёт в предложения.

Ты увидишь предложение в `/admin` → «Предложения», с кнопками принять
или отклонить. Если настроена `LOG_GROUP`, туда же придёт уведомление.
Автор предложения получит короткое сообщение о решении.

Чтобы никто не мог завалить очередь спамом, у каждого человека может
висеть не больше трёх предложений без ответа одновременно.

## 🖼 Баннер и видео

По умолчанию во время звука чёрный экран, это нормально. Но можно
добавить картинку, гифку или даже настоящее видео.

**Баннер**, картинка вместо чёрного экрана:
- `/admin` → «Баннер», или `/banner ссылка`, или ответом на фото, гифку
  или видео команда `/banner`
- `/banner off` — убрать

У YouTube-треков своя обложка подставляется автоматически, даже без
твоего баннера. Если хочешь, чтобы баннер оставался одним и тем же и
не менялся на обложки треков, закрепи его: `/banner pin`, откреплять
обратно `/banner unpin`. Короткие гифки и видео крутятся по кругу, пока
играет звук.

**Настоящее видео**, с движением, не просто картинка:
- `/playvideo название_или_ссылка` — видео с YouTube целиком
- Ответь на видео-сообщение командой `/playvideo`

## 📊 Статистика

`/admin` → «Статистика» показывает: сколько времени бот работает,
сколько треков сыграно за сессию, что играло последним, сколько
предложений ждут решения, сколько сейчас в очереди, сколько админов,
и кто чаще всего заказывает треки. Обнуляется при перезапуске бота.

## 🏗 Деплой на сервер

### Ffmpeg на сервере

Локально ffmpeg ты ставишь сам руками, а вот на сервере (Heroku, Railway,
Render и подобные) его часто нет, и обычным `apt install` там ничего не
поставить. Специально для этого в проекте есть `static-ffmpeg` из
requirements.txt: при первом запуске бот сам скачивает нужный бинарник и
подключает его, никаких дополнительных действий не нужно. Если у тебя
уже есть свой ffmpeg в системе, эта библиотека его не трогает.

Если по какой-то причине сервер не даёт скачать файл с GitHub, в проекте
есть `Dockerfile`, который ставит ffmpeg через apt при сборке образа.

### Heroku

1. Создай приложение на https://heroku.com, подключи GitHub-репозиторий
2. Settings → Config Vars, добавь все переменные из `.env`
3. Resources, включи dyno worker

Если разворачиваешь через Container Registry, `Dockerfile` уже готов:
```bash
heroku container:push worker
heroku container:release worker
```

### Railway, Render и похожие

Просто подключи репозиторий, добавь переменные окружения из `.env` в
настройках проекта, и укажи `python3 main.py` как команду запуска
(или используй готовый `Dockerfile`, если платформа его поддерживает).

## 🔌 Если радио вдруг замолкает

Telegram иногда сам закрывает голосовой чат, например, если решит, что
он неактивен, или из-за временного сетевого сбоя. Раньше бот в этом
случае просто останавливался навсегда. Теперь он:

1. Замечает обрыв и через несколько секунд сам переподключается, играет
   дальше с того же места, где остановился
2. Плюс раз в 45 секунд фоновый сторож проверяет, что всё действительно
   играет, на случай если событие об обрыве до бота вообще не долетело

Сторож смотрит только на то, подключены ли мы к голосовому чату вообще,
а не на то, что именно там происходит. Это важно для длинных видео: пока
трек буферизуется или стоит на паузе, это нормальное состояние, а не
разрыв, и сторож его не трогает. Раньше он ошибочно принимал такие
моменты за обрыв и перезапускал видео с начала, теперь так не будет.

Если обрывы происходят слишком часто подряд (больше пяти раз за пять
минут), бот перестаёт пытаться сам и просто пишет в лог, что дело,
похоже, серьёзнее обычного сетевого сбоя, например юзербота выгнали из
чата или сессия слетела. Тогда стоит посмотреть логи и разобраться руками.

Если звук временами плохой или дёргается, чаще всего дело в мощности
сервера, кодирование звука и особенно видео требует процессора. Если
используешь баннер или видео, `/banner off` заметно снижает нагрузку,
видео весит намного больше, чем просто звук.

## ❓ Частые проблемы

**«ModuleNotFoundError: No module named 'pyrogram'»**
→ `pip uninstall pyrogram -y && pip install -r requirements.txt`
→ Проверь, что venv активирован в том же терминале, где ставишь пакеты

**«ffprobe not installed» или «No such file or directory: 'ffprobe'»**
→ На сервере нет ffmpeg. Обнови зависимости: `pip install -r requirements.txt`
→ static-ffmpeg скачает нужный бинарник сам при следующем запуске
→ Если сервер режет соединения к GitHub, собери через приложенный `Dockerfile`

**Радио время от времени останавливается само**
→ Это должно чиниться само, смотри раздел «Если радио вдруг замолкает» выше
→ Если в логах видно «слишком много попыток», проверь права юзербота в чате

**«Не удалось войти в голосовой чат», `CHAT_ADMIN_REQUIRED`**
→ Юзербот должен быть админом чата с правом «Управление видеочатами»

**«FloodWait»**
→ Telegram временно ограничил запросы, бот подождёт и продолжит сам

**Трек с YouTube не воспроизводится**
→ `pip install -U yt-dlp`, YouTube часто меняет свой API
